# Operación Taller V1

Guía operativa para usar Gmtch Tune OS en un taller real. El objetivo es mantener trazabilidad desde la recepción hasta la entrega comercial.

## Regla principal

El cierre técnico no es cierre comercial.

- Cierre técnico: el trabajo técnico quedó finalizado, con evidencia y post escritura OK si aplica.
- Cierre comercial: la orden fue cobrada y entregada al cliente.

File Service no debe cerrar pagos ni entregas. Órdenes de trabajo maneja cobro, entrega y cierre comercial.

## Flujo operativo

1. Iniciar sesión
   - Cada persona debe entrar con su usuario.
   - No compartir claves.

2. Revisar Dashboard
   - Ver caja, órdenes activas, File Service activos, alertas y checklist operativo.
   - Priorizar trabajos con alertas o pendientes de pago/entrega.

3. Recepción rápida por patente
   - Ingresar patente y buscar.
   - Si existe: revisar vehículo, cliente, últimas visitas y órdenes activas.
   - Si no existe: crear cliente, vehículo y orden.

4. Crear orden
   - Registrar kilometraje, servicio solicitado, síntomas, monto y prioridad.
   - La orden queda como base del trabajo operativo.

5. Subir fotos/evidencia
   - Asociar fotos a la orden.
   - Clasificar por tipo: frontal, tablero, motor, scanner, daño, entrega u otro.

6. Diagnóstico
   - Registrar diagnóstico con DTC.
   - Si no hay DTC, marcar SIN DTC y dejar observación.
   - Adjuntar evidencia de scanner cuando corresponda.

7. File Service si corresponde
   - Crear File Service asociado a la orden.
   - Seleccionar tipo de servicio.
   - Subir archivo original.
   - Asignar responsables si aplica.

8. Procesamiento externo
   - Registrar uso manual de Alientech Recode, KESS3, StageX u otra herramienta.
   - Guardar estado, responsable, observación y archivo resultado si existe.

9. MOD y notificación
   - Subir MOD V1, V2 o V3 según corresponda.
   - Avisar Master o Slave por WhatsApp solo cuando sea necesario.
   - Las notificaciones internas no reemplazan la comunicación crítica.

10. Post escritura
    - Registrar resultado post escritura.
    - Para finalizar técnico debe existir post escritura OK y evidencia suficiente.

11. Finalizar técnico
    - Marca el File Service como finalizado técnicamente.
    - La orden pasa a lista para entrega.
    - No cobra, no entrega y no cierra comercialmente.

12. Confirmar pago y entregar
    - Desde Órdenes, revisar monto total.
    - Confirmar medio de pago, monto pagado y observaciones.
    - Usar “Confirmar pago y entregar”.

13. Revisar ficha vehículo 360
    - Consultar historial del vehículo.
    - Revisar órdenes, diagnósticos, fotos, File Service, pagos y entregas.

## Buenas prácticas

- Registrar responsables por etapa.
- No avanzar sin evidencia mínima.
- Archivar antes que eliminar.
- Usar observaciones claras y cortas.
- Revisar dashboard antes de entregar un vehículo.
