# Plan Maestro Gmtch Tune OS

## Qué es Gmtch Tune OS

Gmtch Tune OS es una plataforma operativa para ordenar el trabajo real de GMTCH: clientes, vehículos, órdenes, diagnóstico, evidencia, File Service, responsables, pagos, entregas, feedback y trazabilidad.

Su foco actual no es vender software hacia afuera, sino estabilizar la operación interna del taller y convertir el conocimiento diario en datos confiables.

## Objetivos

### Objetivo actual: operación interna GMTCH

Usar la plataforma como fuente oficial para recepción, diagnóstico, File Service, seguimiento técnico, cobro, entrega e historial.

### Objetivo siguiente: marcha blanca controlada

Operar con clientes reales, detectar fricción, corregir bugs, limpiar datos demo y validar si el flujo es suficientemente claro para el equipo.

### Objetivo futuro: SaaS para talleres

Replicar el sistema en otros talleres con multiempresa, permisos por organización, planes, reportes avanzados y portales externos controlados.

## Principios del sistema

- La plataforma es la fuente oficial.
- WhatsApp es canal de entrada, no sistema principal.
- Cierre técnico no es cierre comercial.
- No borrar historial operativo.
- Usuarios se desactivan, no se eliminan físicamente.
- Datos demo o test no cuentan en estadísticas reales.
- Responsables indican quién tiene la pelota ahora.
- Cada etapa debe dejar evidencia, fecha y responsable cuando sea posible.

## Módulos actuales

- Clientes CRM básico.
- Vehículos y ficha 360.
- Órdenes de trabajo.
- Diagnóstico.
- Fotos y evidencia.
- File Service.
- Procesamiento externo manual.
- Responsables por etapa.
- Feedback de operarios.
- Dashboard operativo.
- Notificaciones internas.
- Usuarios y roles.
- Exclusión demo de estadísticas.
- Categoría de cliente y prioridad sugerida.

## Módulos futuros

- Reportes avanzados.
- Portal externo para masters o talleres aliados.
- Garantías y reclamos.
- Historial por operario.
- Permisos más finos.
- Automatización WhatsApp.
- Multiempresa.
- Planes SaaS y pagos recurrentes.

## Riesgos actuales

- Datos demo mezclados con datos reales.
- Campos nuevos que dependen de columnas defensivas en producción.
- Flujo real todavía no validado con suficiente volumen.
- WhatsApp puede seguir capturando información que no entra al sistema.
- Reportes pueden ser engañosos si el equipo no registra datos de forma consistente.
