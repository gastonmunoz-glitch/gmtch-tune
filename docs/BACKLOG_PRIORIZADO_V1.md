# Backlog Priorizado V1

Este backlog ordena los próximos trabajos según impacto operativo, riesgo y momento recomendado. La prioridad puede cambiar después de la marcha blanca.

## Prioridad Alta

| Trabajo | Impacto | Riesgo | Momento recomendado |
| --- | --- | --- | --- |
| Validar marcha blanca real | Alto: confirma si el flujo funciona con clientes reales | Medio: aparecerán errores no vistos en pruebas | Inmediato |
| Corregir bugs detectados en uso real | Alto: evita fricción diaria | Bajo/medio según módulo | Durante marcha blanca |
| Mejorar estadísticas reales | Alto: permite entender caja, ventas y carga | Medio: requiere datos limpios | Después de limpiar demo |
| Feedback operarios en ficha y reportes | Alto: captura pendientes y aprendizaje técnico | Bajo: es informativo | Corto plazo |
| Estadísticas de ventas y flujo | Alto: ayuda a tomar decisiones comerciales | Medio: depende de registros consistentes | Corto plazo |
| Fallas recurrentes por vehículo/marca/modelo/DTC | Alto: valor técnico diferencial | Medio: requiere diagnóstico bien ingresado | Corto plazo |
| Portal externo File Service limitado | Alto: abre operación con aliados | Alto: requiere aislamiento y permisos | Después de estabilizar interno |

## Prioridad Media

| Trabajo | Impacto | Riesgo | Momento recomendado |
| --- | --- | --- | --- |
| Portal para masters frecuentes | Alto para File Service | Alto si ven datos internos | Luego de portal externo limitado |
| Automatización WhatsApp como canal de entrada | Medio/alto: reduce copia manual | Alto: puede duplicar o ensuciar datos | Después de ordenar formularios |
| Reportes PDF | Medio: útil para clientes y auditoría | Bajo/medio | Cuando los datos sean confiables |
| Control de garantías/reclamos | Alto en postventa | Medio | Tras validar entregas reales |
| Historial por operario | Medio/alto: mide carga y desempeño | Medio: debe evitar uso punitivo | Después de responsables V1 |
| Permisos más finos | Alto para escalar | Medio/alto | Antes de usuarios externos |

## Prioridad Baja / Futuro

| Trabajo | Impacto | Riesgo | Momento recomendado |
| --- | --- | --- | --- |
| Multiempresa | Muy alto para SaaS | Alto | Etapa SaaS |
| Planes SaaS | Alto comercial | Medio/alto | Después de multiempresa |
| Pagos recurrentes | Alto comercial | Alto | Después de planes |
| App móvil | Medio/alto | Alto por mantenimiento | Cuando web esté estable |
| Integraciones avanzadas | Variable | Alto | Solo con flujo maduro |

## Criterio de avance

Primero se estabiliza operación real GMTCH. Luego se mide. Después se automatiza. Finalmente se abre a terceros.
